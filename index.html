Domain Driven Design - basics

Agenda

  • A common language

    The terms and relationships that reflect domain insight provide the semantics for a language precise enough for technical development.

  • Meaningful code and relevant models

    Code should be an expression of the domain model so it supports the common language.

  • Building blocks

    Layered architecture, entities, value objects and services.

A common language

  • Domain jargon translated to technical dialects, and viceversa, blunts communication.
  • Use the same language in speech and writing.
  • Use the model as the backbone of a common language; a change to the language is a change to the model.

Meaningful code and relevant models

  • Analysis produces a model which captures fundamental concepts from the domain.
  • Design especifies a set of programmable and deployable components.
  • When the code does not easily map to the model correctness of the software is suspect; analysis and design are divided.
